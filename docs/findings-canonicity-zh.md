# 发现 —— 召回差 (Recall-gap)：古典中文 LLM 的抗污染排名

*配套阅读：[`leaderboard.md`](../leaderboard.md)。数据：10 个模型 × 6 个任务 × 100 题，全部基于已存预测的确定性重打分（无新 API 调用）。2026-05。*

## 一句话

绝大多数 "古典中文 LLM" 评测把两件事混在一起测：**真正处理古文的能力** 和 **预训练阶段对古典文献的接触深度**。本工作把它们拆开 —— 按 **文献规范性 tier (canonicity tier)** 给评测题分桶 (T3 = 核心经典如 论语/史记/诗经，几乎被所有现代 LLM 逐字记诵；T1 = 冷门正史如 北齐书/陈书，或非常见诸子)，再以 **Gap = T3 − T1** 作为模型的 "召回依赖度" 指标。

由此引出 3 个发现：

1. **排名并非不变。** 按 T1（抗污染轴）排序，开源中文模型反居前列：Qwen3.5-35B-A3B `0.546`、glm-5 `0.514`，均高于所有 Claude。
2. **Gap 是模型属性，不是评测属性。** 同一 Claude 家族，Gap 范围从 `+0.027` (Haiku 4.5) 到 `+0.220` (Opus 4.7) —— 与 Avg 总分无关。Haiku 是我们测过的能力对齐度最高的排名者，尽管它整体最弱。
3. **`idiom-source`（识别典故出处）任务承担了 70%+ 的污染信号。** 在 `idiom-source` 上，ρ(canonicity tier, 题目难度) = +0.68；在其他 5 个任务上 ρ ≈ +0.06–0.08。剔除 `idiom-source` 后，整套评测在结构上即是抗污染的。

## 方法

对评测中的每道题，按其来源文献打 canonicity tier 标签：

- **T3** —— 核心经典：论语、史记、诗经、大学、中庸 等。在任何万亿 token 级预训练语料中均被假定逐字记诵。
- **T2** —— 知名但非泛滥：后汉书、三国志、春秋三传、资治通鉴 部分章节 等。
- **T1** —— 冷门正史或非常见诸子：北齐书、周书、陈书、隋书 部分章节 等。被大量上采样的可能性较低。

定义机械化地从 `_source` 与 `book` metadata 推出，见 `scripts/canonicity.py`。然后报告每个模型在 **T3 子集 Avg**、**T1 子集 Avg**、**Gap** 三列。

## 结果

按 **T1（抗污染列）** 排序。点估计，每个 tier 每个模型 25–40 题 —— 方向性指标，未做显著性。

- Qwen3.5-35B-A3B：T3 0.656 / T1 0.546 / **Gap +0.110**
- glm-5：T3 0.633 / T1 0.514 / **Gap +0.119**
- claude-opus-4-7：T3 0.732 / T1 0.512 / **Gap +0.220** ← 总分 #1，但冷门 #3
- claude-sonnet-4-6：T3 0.663 / T1 0.507 / **Gap +0.155**
- claude-opus-4-7-thinking：T3 0.696 / T1 0.507 / **Gap +0.188**
- deepseek-3.2：T3 0.643 / T1 0.491 / **Gap +0.151**
- minimax-m2.1：T3 0.616 / T1 0.466 / **Gap +0.150**
- qwen3-coder-next：T3 0.599 / T1 0.455 / **Gap +0.144**
- minimax-m2.5：T3 0.566 / T1 0.445 / **Gap +0.121**
- claude-haiku-4-5：T3 0.461 / T1 0.434 / **Gap +0.027**

总分第一的模型 Gap 最大。Gap 最小的是总分最差的 —— 但是 Haiku 的 T1 (0.434) 与 MiniMax m2.5 的 T1 (0.445) 仅差 0.012，而在经典重的一端差距悬殊 (0.566 vs 0.461)。**任务一旦不再奖励记忆，Haiku 就能追上来。**

## 解读

LLM 评测中所谓 "污染" 通常被当作二元缺陷处理 —— 要么模型见过测试集，要么没见过。数据建议另一种更有用的视角：**所有模型都见过经典；不同的是它们对 "见过" 的依赖程度**。

- Gap 高的模型在任何能依靠记忆的地方都倾向于检索已记诵的文本。在工程意义上有用（更博学的助手），但凡评测中名句占比高，这就被夸大成"古文能力强"。
- Gap 低的模型在冷门文献上和经典上表现接近 —— 其分数反映的是 **古文能力**，而不是 **记忆召回**。这才是一个评测应该测量的区间。

由此推论：**换一个 headline 选择就是换一个排名** —— 两个排名都站得住脚，但回答的是不同问题：

- *哪个模型对古典中文懂得最多？* → 按 **总分 Avg** (Opus 4.7)
- *哪个模型在没被过度训练的古文领域上最强？* → 按 **T1 Avg** (Qwen3.5-35B-A3B)

绝大多数评测只公布前者。我们两者都公布。

## 不主张什么

- **canonicity-tier 是代理变量，不是训练语料覆盖度的直接测量。** 与覆盖度相关但无法直接证明。诚实的陈述是 "冷门文献更难被记忆" —— 外部无从验证某个具体模型的预训练数据。
- **Gap 信号集中在 `idiom-source` 上。** 其他 5 个任务的 ρ(canonicity, 难度) ≈ 0.06–0.08 —— 结构上即抗污染。如果从 Avg 中剔除 `idiom-source`，所有模型的 Gap 均会塌陷。上面的发现更准确的读法是 "本评测有一个 召回重 的任务和五个 能力重 的任务，这种不对称暴露了每个模型的召回依赖度"。
- **n 小。** 10 个模型，每任务 100 题。每个 tier 25–40 题。Gap 差异 < ~0.05 视为噪声。

## 为什么这件事重要

canonicity 分层是一个 **免费的回溯分析**：不需新调用 API，不需人工标注，机械地从语料 metadata 推出。任何采样自公开文献且具有清晰 canonicity 梯度的评测都可以这么做。**最便宜的抗污染防线不是建一个私有 holdout 集（昂贵、永远不可复现），而是用你已经拥有的暴露代理给分数打梯度。**
